## 二、面对对象 (OOP, Object-Oriented Programming) 相关：

### （一）封装继承多态

**Q: 讲一下封装。**<br>
A: <br>
封装是将属于同一业务的数据和方法组合在一起，隐藏内部实现细节，只暴露必要的接口。

**Q: 讲一下继承。**<br>
A: <br>
继承是子类可以继承父类的属性和方法，从而实现代码的复用。

**Q: 讲一下多态。**<br>
A: <br>
多态是指同一个接口可以被不同的对象以不同的方式实现，具体形式是不同类的实例对象调用同名方法时执行动作不同，提高了代码的灵活性和可扩展性。


### （二）`__init__`和`__new__`

**Q: 讲一下`__init__`和`__new__`的区别。**<br>
A: <br>
- `__new__`是用于创建对象实例的方法，负责在内存中申请空间，接收类作为第一个参数，并传递参数；而`__init__`是用于初始化对象实例的方法，接收实例作为第一个参数，将传入的参数绑定到对象实例上。

- `__new__`在对象创建时调用，而`__init__`在对象初始化时调用。

- `__new__`必须返回一个对象实例，而`__init__`不需要返回值。

- 通常情况下，我们只需要重写`__init__`方法来初始化对象实例，只有在需要控制对象创建过程（实现单例模式）、继承修改不可变类型 (tuple, str) 时才需要重写`__new__`方法。


### （三）设计模式

**Q: 讲一下设计模式。**<br>
A: <br>
面向对象提供了对象、封装、继承、多态等机制，设计模式则是面向对象的补充。

设计模式是如何利用这些机制设计构建一套能被反复使用的代码的最佳实践经验总结。

使用设计模式可以提高代码的可维护性、可扩展性和复用性。


**Q: 讲一下单例模式，以及 Python 中其实现方式。**<br>
A: 单例模式是一种设计模式，能确保一个类在项目运行的全生命周期中只有一个实例，并提供一个全局访问点。

Python 中实现单例模式的方式有多种，常见的有以下几种：
1. 使用模块：Python 模块本身就是单例的，可以直接使用模块来实现单例模式。
2. 使用`__new__`类变量方法：通过在类中定义一个类变量来存储实例，并在 `__new__` 方法中检查是否已经存在实例，如果存在则返回该实例，否则创建一个新的实例。

```python
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super(Singleton, cls).__new__(cls, *args, **kwargs)
        return cls._instance
```

3. 使用装饰器：通过定义一个装饰器来实现单例模式。
```python
def singleton(cls):
    instances = {}

    def get_instance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]

    return get_instance

@singleton
class MyClass:
    pass
```

4. 使用元类重写`__call__`方法：通过定义一个元类来实现单例模式。
```python
class SingletonMeta(type):
    _instance = None

    def __call__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super(SingletonMeta, cls).__call__(*args, **kwargs)
        return cls._instance

class Singleton(metaclass=SingletonMeta):
    pass
```


### （四）元类

**Q: 讲一下元类。**<br>
A:<br>
Python 中一切皆对象，类也是对象，元类就是用来创建类的对象的，类似于“机床”。

类默认继承`type`元类，我们可以通过自定义元类，控制类的创建过程，动态地修改类的属性和方法，实现一些特殊的功能。

默认由`type`创建类，定义元类需要继承`type`，重写`__new__`或`__init__`方法来实现自定义的类创建逻辑。继承自定义元类时需要指定`metaclass`参数。

举一个简单的例子，定义一个元类来自动为类添加一个`greet`方法：

```python
class GreetMeta(type):
    def __new__(cls, name, bases, attrs):
        attrs['greet'] = lambda self: f"Hello, I am {self.__class__.__name__}!"
        return super().__new__(cls, name, bases, attrs)

class MyClass(metaclass=GreetMeta):
    pass
```


### （五）面向切面编程 (AOP, Aspect-Oriented Programming)

**Q: 讲一下什么是面向切面编程。**<br>
A: <br>
面向切面编程是一种编程范式，也是面向对象机制的补充。面向对象可以解决父类子类这种有极强关联的类间的关系。

面向切面编程则为多个毫无关联的类，添加相同的通用功能，将横切关注点（如日志记录、权限校验、性能监控、数据库事务管理等）与核心业务逻辑分离，使得代码更加模块化和可维护。

装饰器机制是面向切面编程的一种实现方式。


### （六）装饰器 (decorator)

**Q: 讲一下装饰器。**<br>
A: <br>
装饰器的本质是一个高阶函数，Python 中函数本身可以被当作变量传递和返回，装饰器实际上也就是一个*接收函数作为参数，并返回一个新函数的可调用对象*。

实际开发中的作用是无需修改函数具体的内部实现即可添加功能，提高了代码模块化程度，方便开发和后续维护。

    return wrapper

### （七）可迭代对象、迭代器、生成器

**Q: 讲一下什么是可迭代对象，以及迭代器和生成器的区别。**<br>
A: <br>
- 可迭代对象是实现了 `__iter__` 方法的对象，可以使用 `for` 循环进行遍历。可迭代对象本身只是一种数据集合的容器，它自己并不负责记录遍历的状态或位置。
- 迭代器是实现了 `__iter__` 和 `__next__` 方法的对象，可以通过 `next()` 函数获取下一个元素，直到没有元素时抛出 `StopIteration` 异常。
- 生成器是使用 `yield` 关键字定义的函数，返回一个迭代器对象，当函数执行到 yield 时，会暂停执行，保存当前的所有的运行信息（包括局部变量和指令指针），并将 yield 后面的值返回给调用者。当下一次调用 next() 时，函数会从上次挂起的地方恢复执行。
    - 在后端开发处理海量数据库查询，或者在 Agent 开发中处理 LLM 大模型的流式 Token 响应时，生成器能将内存占用从 $O(N)$ 降低到 $O(1)$，是解决性能瓶颈的关键技术。

## 三、多进程、多线程相关

### （一）进程、线程概念

**Q: 讲一下什么是进程。**<br>
A: <br>
进程是操作系统分配资源的最小单位。拥有独立的堆栈和数据段，进程间切换开销极大，但隔离性强，能利用多核 CPU 实现真正并行。


**Q: 讲一下什么是线程。**<br>
A: <br>
线程是操作系统调度执行的最小单位，依附于进程存在。同进程下的线程共享堆内存，上下文切换开销小。

**Q: 什么情况下使用多进程，什么情况下使用多线程？**<br>
A: <br>
- 如果业务属于**CPU 密集型**（如图像处理、矩阵运算），推荐使用 multiprocessing 多进程。
- 如果属于 **I/O 密集型**（如网络请求、数据库查询），推荐使用 threading 多线程结合队列（Queue）来实现。

## 七、深浅拷贝

**Q: 讲一下什么是拷贝？**<br>
A: <br>
在数据传递使用过程中，可能会发生数据被修改的问题。为了防止数据被修改，就需要在传递一个副本，即使副本被修改，也不会影响原数据的使用，生成副本的过程就叫做拷贝。根据拷贝的方式不同，分为浅拷贝和深拷贝.

**Q: 讲一下直接赋值、浅拷贝和深拷贝的区别？**<br>
A: <br>

1. 直接赋值<br>
    直接赋值是将一个变量指向另一个变量的内存地址，引用计数+1。两个变量指向同一个对象，修改其中一个变量会影响另一个变量。

2. 深浅拷贝<br>
    深浅拷贝的区别体现在子对象的处理上：是否递归地复制对象中的元素

    - 浅拷贝是**创建一个新的对象**，在内存中为新对象开辟一片新位置，但其中的元素仍然是对原对象中元素的引用。

        对于不可变类型（如字符串、整数等），浅拷贝会创建一个新的对象；

        对于可变类型（如列表、字典等），浅拷贝会创建一个新的对象，但其中的元素仍然是对原对象中元素的引用。修改其中一个变量会影响另一个变量。

    - 深拷贝<br>
        深拷贝是**创建一个新的对象**，在内存中为新对象开辟一片新位置，并且**递归**地复制原对象中的所有元素，为子对象的元素也生成一个完全独立的对象，在内存中开辟新空间。

        修改其中一个变量不会影响另一个变量。


**Q: Python 为什么默认使用浅拷贝？**<br>
A: <br>
浅拷贝的速度快、空间占用少、效率更高。对于大多数情况下，浅拷贝已经足够满足需求，深拷贝会带来额外的性能开销。因此，Python 默认使用浅拷贝来提高性能，只有在需要完全独立的对象时才使用深拷贝。

浅拷贝适用于仅需复制顶层结构的场景，如列表、字典等容器对象的复制，或者当对象中包含不可变类型时，浅拷贝已经足够满足需求。

当需要完全独立的副本时，如配置文件修改、复杂数据结构处理等，才需要使用深拷贝来确保数据的独立性和安全性.
