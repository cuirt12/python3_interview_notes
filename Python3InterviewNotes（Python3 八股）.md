## 二、面对对象 (OOP, Object-Oriented Programming) 相关：

### （一）封装继承多态

**Q: 讲一下封装。**<br>
A: <br>
封装是将属于同一业务的数据和方法组合在一起，隐藏内部实现细节，只暴露必要的接口。

---

**Q: 讲一下继承。**<br>
A: <br>
继承是子类可以继承父类的属性和方法，从而实现代码的复用。

---

**Q: 讲一下多态。**<br>
A: <br>
多态是指同一个接口可以被不同的对象以不同的方式实现，具体形式是不同类的实例对象调用同名方法时执行动作不同，提高了代码的灵活性和可扩展性。


### （二）`__init__` 和 `__new__`

**Q: 讲一下 `__init__` 和 `__new__` 的区别。**<br>
A: <br>
- `__new__` 是用于创建对象实例的方法，负责在内存中申请空间，接收类作为第一个参数，并传递参数；而 `__init__` 是用于初始化对象实例的方法，接收实例作为第一个参数，将传入的参数绑定到对象实例上。

- `__new__` 在对象创建时调用，而 `__init__` 在对象初始化时调用。

- `__new__` 必须返回一个对象实例，而 `__init__` 不需要返回值。

- 通常情况下，我们只需要重写 `__init__` 方法来初始化对象实例，只有在需要控制对象创建过程（实现单例模式）、继承修改不可变类型 (tuple, str) 时才需要重写 `__new__` 方法。


### （三）设计模式

**Q: 讲一下设计模式。**<br>
A: <br>
面向对象提供了对象、封装、继承、多态等机制，设计模式则是面向对象的补充。

设计模式是如何利用这些机制设计构建一套能被反复使用的代码的最佳实践经验总结。

使用设计模式可以提高代码的可维护性、可扩展性和复用性。

---

**Q: 讲一下单例模式，以及 Python 中其实现方式。**<br>
A: 单例模式是一种设计模式，能确保一个类在项目运行的全生命周期中只有一个实例，并提供一个全局访问点。

Python 中实现单例模式的方式有多种，常见的有以下几种：
1. 使用模块：Python 模块本身就是单例的，可以直接使用模块来实现单例模式。
2. 使用 `__new__` 类变量方法：通过在类中定义一个类变量来存储实例，并在 `__new__` 方法中检查是否已经存在实例，如果存在则返回该实例，否则创建一个新的实例。

```python
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super(Singleton, cls).__new__(cls, *args, **kwargs)
        return cls._instance
```

3. 使用装饰器：通过定义一个装饰器来实现单例模式。
```python
def singleton(cls):
    instances = {}

    def get_instance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]

    return get_instance

@singleton
class MyClass:
    pass
```

4. 使用元类重写`__call__`方法：通过定义一个元类来实现单例模式。
```python
class SingletonMeta(type):
    _instance = None

    def __call__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super(SingletonMeta, cls).__call__(*args, **kwargs)
        return cls._instance

class Singleton(metaclass=SingletonMeta):
    pass
```


### （四）元类

**Q: 讲一下元类。**<br>
A:<br>
Python 中一切皆对象，类也是对象，元类就是用来创建类的对象的，类似于“机床”。

类默认继承 `type` 元类，我们可以通过自定义元类，控制类的创建过程，动态地修改类的属性和方法，实现一些特殊的功能。

默认由 `type` 创建类，定义元类需要继承 `type`，重写 `__new__` 或 `__init__` 方法来实现自定义的类创建逻辑。继承自定义元类时需要指定 `metaclass` 参数。

举一个简单的例子，定义一个元类来自动为类添加一个 `greet` 方法：

```python
class GreetMeta(type):
    def __new__(cls, name, bases, attrs):
        attrs['greet'] = lambda self: f"Hello, I am {self.__class__.__name__}!"
        return super().__new__(cls, name, bases, attrs)

class MyClass(metaclass=GreetMeta):
    pass
```


### （五）面向切面编程 (AOP, Aspect-Oriented Programming)

**Q: 讲一下什么是面向切面编程。**<br>
A: <br>

面向切面编程则为多个毫无关联的类，添加相同的通用功能，将横切关注点（如日志记录、权限校验、性能监控、数据库事务管理等）与核心业务逻辑分离，使得代码更加模块化和可维护。

装饰器机制是面向切面编程的一种实现方式。


### （六）装饰器 (decorator)

**Q: 讲一下装饰器。**<br>

```python
    # 命名规范：wrapper

        #
        #

    return wrapper

@time_logger

    return

```

### （七）可迭代对象、迭代器、生成器

**Q: 讲一下什么是可迭代对象，以及迭代器和生成器的区别。**<br>
A: <br>
- 可迭代对象是实现了 `__iter__` 方法的对象，可以使用 `for` 循环进行遍历。可迭代对象本身只是一种数据集合的容器，它自己并不负责记录遍历的状态或位置。
- 迭代器是实现了 `__iter__` 和 `__next__` 方法的对象，可以通过 `next()` 函数获取下一个元素，直到没有元素时抛出 `StopIteration` 异常。
- 生成器是使用 `yield` 关键字定义的函数，返回一个迭代器对象，当函数执行到 `yield` 时，会暂停执行，保存当前的所有的运行信息（包括局部变量和指令指针），并将 `yield` 后面的值返回给调用者。当下一次调用 `next()` 时，函数会从上次挂起的地方恢复执行。
    - 在后端开发处理海量数据库查询，或者在 Agent 开发中处理 LLM 大模型的流式 Token 响应时，生成器能将内存占用从 $O(N)$ 降低到 $O(1)$，是解决性能瓶颈的关键技术。

## 三、多进程、多线程相关

### （一）进程、线程、协程概念

**Q: 讲一下什么是进程。**<br>
A: <br>
进程是操作系统分配资源的最小单位。拥有独立的堆栈和数据段，进程间切换开销极大，但隔离性强，能利用多核 CPU 实现真正并行。

---

**Q: 讲一下什么是线程。**<br>
A: <br>
线程是操作系统和 CPU 调度执行的最小单位，依附于进程存在。同进程下的线程共享内存，上下文切换开销小。

---

**Q: 讲一下什么是协程。**<br>
A: <br>
协程是一种用户态的轻量级线程，由程序员控制其执行和切换。协程在执行过程中可以主动让出控制权，从而实现并发执行。协程相比线程，带有一定的CPU上下文信息，因此切换起来开销更小。

---

**Q: 讲一下什么是上下文管理器。**<br>
A: <br>
上下文管理器是一种内部实现了 `__enter__` 和 `__exit__` 方法的对象，用于管理资源的安全获取和释放。常用于文件操作、数据库连接等需要确保资源正确释放的场景。使用 `with` 语句可以简化上下文管理器的使用，确保即使发生异常也能正确释放资源。

---

**Q: 讲一下什么是线程安全。**<br>
A: <br>
线程安全指的是在多线程环境下，多个线程访问同一资源时，不会发生数据竞争和不一致的情况。线程安全的代码能够正确处理多个线程同时访问共享资源的情况，确保数据的完整性和一致性。

---

**Q: 什么情况下使用多进程，什么情况下使用多线程？**<br>
A: <br>
- 如果业务属于**CPU 密集型**（如图像处理、矩阵运算），推荐使用 `multiprocessing` 多进程。
- 如果属于 **I/O 密集型**（特别是网络请求、数据库查询、文件读写），推荐使用 `threading` 多线程结合队列（Queue）来实现。

### （二）并发与并行
并发的概念比较宽泛，指的是一个时间段内，计算机可以同时执行多个任务。
而并行则是严格的，指的是在同一时刻，多个任务同时执行。并发可以通过多线程或异步编程来实现，而并行则需要多进程或分布式计算来实现。

### （三）锁

**Q: 讲一下什么是锁和 Python 中的锁类型。**<br>
A: <br>
锁是并发编程中用于控制对共享资源访问的机制，确保在同一时间只有一个线程或进程能够访问特定资源，防止数据竞争和不一致。

Python 中的 `threading` 模块提供了多种锁类型：
- 最常用的两种锁（状态锁）：
    - 互斥锁 `Lock`：最基本的锁，状态只有“锁定”和“未锁定”。如果一个线程获取了锁，其他试图获取该锁的线程会被阻塞，直到锁被释放。提供了 acquire() 和 release() 方法来控制锁的获取和释放。互斥锁不能连续加锁再连续解锁，否则会引发死锁，导致资源无法被释放，线程阻塞。
    - 可重入锁 `RLock`：允许同一线程多次获取锁，必须要释放相同次数的锁才能真正释放。**函数嵌套调用**，或者**递归函数**（函数自己调用自己），且**这些函数内部都需要访问同一个共享资源**时，我们就必须使用 `RLock` 来避免单线程死锁。
- 其他锁类型：
    - `Semaphore`（信号量）：允许一定数量的线程同时访问一个资源，   通过 acquire() 和 release() 方法来控制访问数量。
    - `Condition`（条件变量）：提供了线程间的通知机制，允许线程在特定条件下等待或被通知继续执行。

3. GIL（全局解释器锁）：CPython 解释器在早期设计时为了保证内存管理的安全以及单线程性能的优化，引入了 GIL（全局解释器锁，Global Interpreter Lock）。
GIL 本身是一个互斥锁，它保证了在任何时刻，只有一个线程能够执行 Python 字节码。这导致 Python 的多线程无法真正利用多核 CPU 的优势来实现并行计算。


### （三）异步

**Q: 讲一下异步的概念。**<br>
A: <br>
异步是一种编程范式，允许程序在等待某些操作完成时继续执行其他任务，而不是阻塞等待。异步编程通过事件循环机制来管理任务的调度和执行，使得程序能够更高效地处理 I/O 密集型任务。

---

**Q: 讲一下 Python 实现异步。**<br>
A: <br>
Python 主要通过 `asyncio` 模块来基于单线程和`协程`实现异步编程。`asyncio` 提供了事件循环、协程和任务等机制，使得开发者可以编写高效的异步代码。当一个线程遇到 I/O 操作时，通过封装了 `yield` 的 `await` 关键字方法释放 GIL，事件循环会将控制权转移给其他任务，直到 I/O 操作完成后再继续执行原来的任务，从而实现了非阻塞的异步执行。


### （四）并行

**Q: 为什么说 Python 的并行计算是伪并行？**<br>
A: <br>
Python 的并行计算是伪并行，主要是因为 Python 的全局解释器锁（GIL）的存在。GIL 保证了在任何时刻，在同一时刻一个进程中只有一个线程能够执行 Python 字节码。这导致 Python 的多线程无法真正利用多核 CPU 的优势来实现并行计算。

---

**Q: 如何实现真并行？**<br>
A: <br>
要实现真正的并行计算，可以使用 `multiprocessing` 模块来创建多个进程，每个进程拥有独立的内存空间和 GIL，这样就可以利用多核 CPU 来实现真正的并行计算。但相应的开销会大很多。


## 七、内存管理机制
内存管理分为：
- 创建 —— 申请与分配机制
- 销毁 —— 垃圾回收机制
申请与分配属于非常底层的知识，垃圾回收机制更重要一些。

**Q: Python 的内存管理机制了解吗？**<br>
A: <br>
对于 Python 内置对象（如整数、字符串等），Python 会使用**小对象池**来优化内存分配和管理。

对于较大的对象，Python 会使用**对象分配器**直接向操作系统申请内存。对象分配器的底层是调用 c 的 `malloc` 等函数来管理内存。

---

**Q: Python 的垃圾回收机制了解吗？**<br>
A: <br>
Python 的垃圾回收机制以**引用计数**为主，以标记-清除和分代回收为辅助。
1. 引用计数：每个对象都有一个引用计数器，记录有多少个引用指向该对象。当引用计数为零时，对象会被立即销毁并释放内存。
    - 但是引用计数无法处理循环引用的情况，即两个或多个对象互相引用，导致它们的引用计数永远不为零，从而无法被垃圾回收器回收。
2. 标记-清除：当引用计数器无法处理循环引用时，比如可能包含其他对象的 `List` 等容器，垃圾回收器会**定期**运行标记-清除算法来检测和清理循环引用的对象。简单来说是基于有向图的一种算法。
    - 第一阶段先从根对象出发遍历标记所有从根对象出发可达的对象；
        - e.g. 根对象是全局变量、当前函数的局部变量、当前执行的函数的参数等。
    - 第二阶段扫描所有对象，清除未被标记的对象，即使他们互相引用。
3. 分代回收：标记清除算法需要进行大量遍历扫描，消耗时间资源，因此引入分代回收。Python 将对象分为三代（0、1、2），新创建的对象属于第 0 代，经过垃圾回收后仍然存在的对象会被提升到下一代。垃圾回收器会优先回收年轻一代的对象，因为大多数对象的生命周期较短。

由于垃圾回收机制极度依赖引用计数，也因此引入了 GIL。


## 七、深浅拷贝

**Q: 讲一下什么是拷贝？**<br>
A: <br>
在数据传递使用过程中，可能会发生数据被修改的问题。为了防止数据被修改，就需要在传递一个副本，即使副本被修改，也不会影响原数据的使用，生成副本的过程就叫做拷贝。根据拷贝的方式不同，分为浅拷贝和深拷贝.

---

**Q: 讲一下直接赋值、浅拷贝和深拷贝的区别？**<br>
A: <br>

1. 直接赋值<br>
    直接赋值是将一个变量指向另一个变量的内存地址，引用计数+1。两个变量指向同一个对象，修改其中一个变量会影响另一个变量。

2. 深浅拷贝<br>
    深浅拷贝的区别体现在子对象的处理上：是否递归地复制对象中的元素

    - 浅拷贝是**创建一个新的对象**，在内存中为新对象开辟一片新位置，但其中的元素仍然是对原对象中元素的引用。

        对于不可变类型（如字符串、整数等），浅拷贝会创建一个新的对象；

        对于可变类型（如列表、字典等），浅拷贝会创建一个新的对象，但其中的元素仍然是对原对象中元素的引用。修改其中一个变量会影响另一个变量。

    - 深拷贝<br>
        深拷贝是**创建一个新的对象**，在内存中为新对象开辟一片新位置，并且**递归**地复制原对象中的所有元素，为子对象的元素也生成一个完全独立的对象，在内存中开辟新空间。

        修改其中一个变量不会影响另一个变量。

---

**Q: Python 为什么默认使用浅拷贝？**<br>
A: <br>
浅拷贝的速度快、空间占用少、效率更高。对于大多数情况下，浅拷贝已经足够满足需求，深拷贝会带来额外的性能开销。因此，Python 默认使用浅拷贝来提高性能，只有在需要完全独立的对象时才使用深拷贝。

浅拷贝适用于仅需复制顶层结构的场景，如列表、字典等容器对象的复制，或者当对象中包含不可变类型时，浅拷贝已经足够满足需求。

当需要完全独立的副本时，如配置文件修改、复杂数据结构处理等，才需要使用深拷贝来确保数据的独立性和安全性.

---

## 十一、异常处理
**Q: Python 的异常处理机制了解吗？**<br>
A: <br>
Python 异常管理分为自动异常和手动异常两种。

自动异常是由 Python 解释器在运行时检测到错误时自动抛出的异常，代码结构为`try-except-else-finally`。
- `try` 块中放置可能引发异常的代码；
- `except` 块中放置处理异常的代码，可以指定捕获特定类型的异常，如 `ZeroDivisionError`、`IndexError` 等，也可以使用通配符捕获所有异常；
- `else` 块中放置没有发生异常时执行的代码；
- `finally` 块中放置无论是否发生异常都会执行的代码。

```python
def calculate(a, b):
    try:
        result = a / b

    except ZeroDivisionError:
        print("Error: Division by zero is not allowed.")
        return None

    else:
        print("Calculation successful.")
        return result

    finally:
        print("Calculation completed.")
```

```python
# 使用 raise 的简单示例：手动抛出异常（参数校验）
def divide_checked(a, b):
    if b == 0:
        raise ValueError("b 不能为 0（手动 raise 的示例）")
    return a / b


# 示例调用
try:
    print(divide_checked(10, 0))
except ValueError as e:
    print("Caught:", e)
```

## 十一、测试框架
**Q: 你们项目中是怎么进行测试的？**<br>
A: <br>
我们项目使用的是目前最流行的 `pytest` 测试框架，主要用于单元测试和集成测试。相比于传统的 `unittest` 框架，`pytest` 提供了更简洁的语法、更强大的功能和更丰富的插件生态，使得编写和维护测试代码更加高效和便捷。

`pytest` 使用的是原生 `assert` 断言，当测试用例出现问题时抛出 `AssertionError` 异常，并提供详细的错误信息，帮助开发者快速定位问题。

主要过程是：
1. 编写测试用例：在项目的 `tests` 目录下编写测试用例，通常以 `test_` 开头命名测试文件和测试函数。
2. 运行测试：使用命令行工具运行测试，例如 `pytest` 命令会自动发现并执行 `tests` 目录下的测试用例，并生成测试报告。
3. 集成测试：在 Agent 或后端接口开发中，我通常会结合 `pytest` 和 `pytest-asyncio` 插件来测试异步代码。

`pytest-asyncio` 可以为测试环境注入 `asyncio` 事件循环，使得我们能够编写和运行基于协程的异步测试用例。

`pytest` 还有一个名为夹具（fixture）的功能，可以用于设置测试环境、提供测试数据等，极大地提高了测试代码的复用性和可维护性。但我没有在项目中使用过夹具，后续可以学习一下。

## 十一、日志管理
**Q: 你们项目中是怎么进行日志管理的？**<br>
A: <br>
日志是用来记录程序运行状态、调试信息和错误信息的重要工具。在我们的项目中，我们使用 Python 内置的 `logging` 模块来进行日志管理。

总的来说就是定义一个日志器`logger`对象，通过 `__name__` ，设置日志级别（如 DEBUG、INFO、WARNING、ERROR、CRITICAL），配置日志格式（如时间戳、日志级别、消息内容等），并将日志输出到控制台或文件中。

```python
def main():
    logging.basicConfig(filename='myapp.log', level=logging.INFO)
    logger.info('Started')
    mylib.do_something()
    logger.info('Finished')

if __name__ == '__main__':
    main()
```
